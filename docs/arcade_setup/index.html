Arcade Setup¶

Nothing is more frustrating than trying to start, only to find you aren’t setup correctly. Let’s write a tiny program to ensure that:

  • Arcade is installed correctly, and…
  • PyCharm can run our small Python program

Objectives¶

  • Install Arcade
  • Run a simple non-Arcade program

Steps¶

  1. Project. Make sure you have the arcade_tutorial project open in PyCharm.

  2. New file. In the PyCharm menu, choose File -> New -> Python File, then name file game.py.

  3. Code. In the new file contents, enter the following:

    1
2
3
4
    import arcade

if __name__ == '__main__':
    print(arcade.RELEASE)

  4. Run. On line 93, in the left gutter, click the green run arrow and choose Run 'game'.

  5. Error. A panel should appear at the bottom with an error message – we haven’t installed arcade yet into this virtual environment!

  6. Install. Click on the red-squigly line for arcade on line one, then click on the red light bulb that pops up. Choose the Install package arcade menu item.

  7. Run. Once installed, click the green play button (either on line 9 or in the Run tool window) to re-run our program.

  8. Output. You now see, in the Run window, the version number.

What’s Going On¶

In this step we ran a Python program. Cool! We used the Python that we setup as the virtual environment for this project, then created a new file called game.py and told that Python to run the file.

Actually, we told PyCharm to tell Python to run it, then PyCharm helped along the way. For example, the output went to a nice output window.

But we had a problem the first time: the Arcade library isn’t installed. PyCharm offered to install this Python package into this project’s virtual environment.
